## Overview
This repository was created for a Computer Security lab focused on learning and applying **OpenSSF Scorecard** and **OpenSSF Best Practices**. The goal of the project is to explore how open source security tools evaluate repositories, identify weaknesses, and guide improvements to repository security posture.

## Purpose
The purpose of this project is to:
- demonstrate the use of OpenSSF security tools on a GitHub repository
- display repository security status using public badges
- identify security findings through Scorecard and code scanning
- improve repository settings and workflow configuration based on those findings

## Lab Objectives
This project supports the following course objectives:
- complete the OpenSSF Scorecard training
- earn the OpenSSF Best Practices badge
- earn the OpenSSF Scorecard badge
- analyze security findings in the repository
- address high-priority issues and improve repository security

## Security Improvements Made
During this lab, the repository was updated to improve its security posture. Changes included:
- configuring branch protection for the `main` branch
- improving GitHub Actions workflow security
- restricting workflow token permissions based on least privilege
- enabling and validating repository security checks through code scanning
